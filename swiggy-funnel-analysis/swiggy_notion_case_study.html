Product case study — UX / funnel analysis
Swiggy Checkout Funnel Optimization
Platform: Swiggy (food delivery) Scope: Cart → Order confirmation Market: India
Problem & objective
Problem: Swiggy's checkout funnel loses high-intent users to compounding trust failures — fee opacity, address errors, and unhandled mid-flow errors — before they ever reach the payment button.

Objective: Increase Cart → Payment conversion by eliminating the three root causes of abandonment, ranked by revenue impact.

Cart Review
Root cause analysis — ranked by impact
#1
Strategic pricing shift with adversarial UX presentation
Swiggy's growth→profitability pivot raised platform fees incrementally, but the UX hides true cost until the last scroll. The delta between browsing price and checkout price is entirely a design choice — not a product necessity.
#2
Over-reliance on third-party map APIs with no user-side validation
The Address Flagger Model (RoBERTa) detects dissonance but then burdens the user with correcting geospatial coordinates on a tiny mobile interface — cognitively the hardest possible fix. Overload leads to abandonment.
#3
Hyperlocal inventory volatility with no predictive recovery flow
The "restaurant closed" error modal is a dead end. Real-time state changes in hyperlocal logistics are inevitable; the failure is architectural — the product hasn't built a graceful recovery path.
#4
Dark pattern UI inflating subscription metrics at trust's expense
Auto-injection of Swiggy One to the cart is documented deceptive architecture. Short-term subscription boosts come at the cost of long-term session trust and brand equity.

Recommendations
01 — Progressive Price Transparency
High impact · Low effort
Problem
Users anchor to menu price; ₹350 final bill vs ₹200 expected creates abandonment at the last scroll.
Insight
Fee opacity is a design choice. The same fees shown progressively during browsing cause no shock.
Fix
Sticky bottom bar on menu page: "Cart: 2 items · Est. total ₹312 (incl. fees)" — updates on each item add.
Expected
>4% lift in C→P conversion. Carts may dip slightly but carry higher purchase intent.
02 — Visual Landmark Anchor Verification
High impact · Med effort
Problem
Users can't correct Geohash-snapped address pins via a tiny draggable map on mobile.
Insight
Visual landmark recognition is 10× faster than abstract spatial mapping for human cognition.
Fix
On RoBERTa flag: replace pin-drag UI with "Are you near [Landmark A] or [Landmark B]?" using nearby POI photos.
Expected
>10% reduction in location-based refund requests; lower support ticket volume.
03 — Predictive Cart Porting
Med impact · High effort
Problem
Hard error modal terminates sessions — CVR drops 90% with no recovery path.
Insight
User intent is captured in the cart. Friction is rebuilding it — not choosing a different restaurant.
Fix
Suppress error modal; auto-port items to nearest equivalent restaurant with one-tap "Accept Alternate" CTA.
Expected
>15% hard error session recovery. Requires tight recommendation accuracy to avoid CSAT drop.
04 — Remove Basket Sneaking
High trust · Low effort
Problem
Auto-injected Swiggy One subscriptions trigger defensive behavior at step 1 of the funnel.
Fix
Audit and remove all cart auto-injection patterns. Replace with explicit opt-in contextual upsells.
Expected
Immediate trust recovery. Short-term subscription metric dip; long-term retention improvement.
What not to prioritize
✕
Slide-to-Pay adjustments — intentional friction is working; removing it increases accidental taps, not conversion.
✕
New payment method integrations — UPI runs at 0.6% decline rate. The bottleneck is upstream.
✕
UPI app-switch crash fixes as P1 — medium confidence; bank server downtime dominates payment failure, not memory leaks.

Conclusion
Swiggy's checkout abandonment is not a payment problem — it is a trust and transparency problem occurring well before the user reaches the pay button. Fee opacity destroys price acceptance, address fragility destroys delivery confidence, and hard error dead-ends destroy session momentum. Fixing these three in order — Progressive Transparency first (highest impact, medium effort), then Landmark Anchor, then Cart Porting — recovers a material portion of the estimated ₹82.5M+ daily revenue lost to checkout abandonment.
